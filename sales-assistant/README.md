# 预测性维护售前助手 MVP

这是一个可直接打开的静态 MVP，用现有 `设备预测性维护问题及解答.docx` 中的 22 条 FAQ 搭建“销售随身售前助手”。

## 使用方式

直接打开：

```text
D:\CodexSpace\PreSalesQA\sales-assistant\index.html
```

无需安装依赖，也不需要启动后端服务。

## 已实现

- 自然语言搜索和分类筛选。
- 客户角色、行业、设备、异议、已有系统的轻量场景表单。
- 每条知识输出四段：一句话回答、展开话术、建议追问、可转发摘要。
- 敏感内容风控提醒：价格、认证、案例、补贴、质保、SLA 不自由编造。
- 销售现场问题提交，进入本地待审核池。
- 知识运营面板：待审核、无答案记录、低满意回答、草稿和上传记录。
- 文本/Markdown FAQ 简单解析为草稿；DOCX 上传先进入待后台解析队列。

## 后续接入真实小程序/LLM

- 将 `data.js` 迁移到后端知识库或 CMS。
- 将 `composeAnswer()` 替换为 RAG + LLM 服务，保留当前敏感字段和风控逻辑。
- 将 localStorage 中的反馈、评分、草稿迁移到企业微信用户体系下的数据库。
- DOCX 解析建议放在后端，用文档解析服务拆分 FAQ、标签和版本信息。
